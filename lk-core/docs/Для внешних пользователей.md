# Управление записями в сервисе через API

Чтобы начать пользоваться сервисом через API, администратор должен создать ключ доступа через соответствующее меню. Этот ключ доступа вам необходимо предоставлять в заголовке при каждом запросе.

Если вы считаете, что ключ был скомпрометирован, свяжитесь с одним из администраторов сервиса для отзыва ключа доступа в получения нового.

Так как содержание форм может отличаться в зависимости от настройки сервиса, в этой документации будет использоваться обозначение `FormContent` для одного объекта записи, [идентично обозначениям типов в TypeScript](https://www.typescriptlang.org/docs/handbook/2/objects.html#the-array-type).

Для уточнения возможных значений полей обратитесь к документации [Описание типов полей](./Описание%20типов%20полей.md).

# Чтение всех существующих записей

```javascript
GET lk-form/api
Headers: {"Auth-Key": ваш_ключ}
Response: FormContent[]
```

# Чтение определённой записи

```javascript
GET lk-form/api/:id
Headers: {"Auth-Key": ваш_ключ}
Response: FormContent
```

# Редактирование записи

```javascript
PUT lk-form/api/:id
Headers: {"Auth-Key": ваш_ключ}
Body: FormContent
Response: FormContent //Изменённый
```

# Добавление новой записи

При добавлении записи, если сервис предполагает добавление отправителя к записи, то сервис может передать идентификатор отправителя в поле `submitter`. Если сервис не предполагает такого, то поле будет проигнорировано.

Также учтите все ограничения полей, которые могут быть наложены в описывающем файле, перед тем, как отправить новую запись. Если запись не удастся добавить, вы получите ошибку с информацией о ней.

```javascript
POST lk-form/api
Headers: {"Auth-Key": ваш_ключ}
Body: {...FormContent, submitter: "YourDearAdmin"}
Response: "OK"
```

# Удаление записи

```javascript
DELETE lk-form/api/:id
Headers: {"Auth-Key": ваш_ключ}
Response: "OK"
```
