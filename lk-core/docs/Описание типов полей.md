# Описание типов полей

Этот документ содержит все поддерживаемые поля формы, которые можно задать через описывающий файл.

## required

Для того, чтобы пометить поле обязательным для заполнения, необходимо передать параметр `required: true` для каждого объекта в списке столбцов формы описывающего файла (`fields[n].formFields`).

Если пометить поле как обязательное, то на странице отправки формы будет добавлен указатель о том, что поле является обязательным. Также сервер будет проверять это поле с помощью функции валидации, которая указана в соответствующем типе в этой документации.

## string

Поле для ввода текста. В зависимости от передаваемого параметра `appearance` может принимать вид небольшого поля для ввода текста `<input type="text">` на половину ширины формы (отсутствие параметра) или вид поля для ввода большого текста `<textarea>` на всю ширину формы (при передаче `appearance: "long"`)

Тип в базе данных: `TEXT`

Функция валидации из class-validator: `isString`

## number

Поле для ввода чисел. 

Тип в базе данных: `NUMERIC`

Функция валидации из class-validator: `isNumber`

## bool

Тип в базе данных: `BOOLEAN`

Функция валидации из class-validator: `isBoolean`

## list

Тип в базе данных: `TEXT[]`

Функция валидации из class-validator: `isArray`

## info

Тип в базе данных: `TEXT`

Функция валидации из class-validator: `isEmpty`

## date

Тип в базе данных: `TIMESTAMPTZ`

Функция валидации из class-validator: `isDate`
